Side Projects

I do some TO'ing now. The process for check-ins was irritating me because it involved exporting an excel spreadsheet from start.gg, and I don't have excel. So I then have to upload the excel spreadsheet to google drive and open it with google sheets, just to copy and paste one column into another google sheet.

Turns out that start.gg does have an API, and the API is pretty in-depth but uses some odd third party API library or something called GraphQL.

The workflow is as follows:

  1. have an auth token.
  2. get the "slug" of the tournament + event you want to pull attendees from. (e.g. tournament/boston-blue-beat-online-weeklies-2025-5-19-to-5/event/5-19-bbcf-pc-bbb-online-2025-weekly)
  3. get the eventID from the slug
  4. get the attendee information from the eventID

There's a little more complexity about it I'll get into on a Tech post. But for now I have a functional script that can make a .csv (just a name per line) of every attendee for a given event that I can just copy and paste into google sheets. Makes the entire process approximately 75% less annoying for me.

I have a few goals to work on:

  • make a UI. (not again!)
  • make it properly use GraphQL so I can populate objects rather than brute forcing through the response JSON
  • make it possible to choose what attributes to print out, thanks to the above point
  • make it easier to get a tournament slug; maybe make it so you paste the tournament name and then the event name and then it fills out the rest, or you provide the entire page URL it and gets the slug from that.

Point 3 exists because people like to set their discord names different from their start.gg names, but because the tournament I'm TO'ing for requires you to have a discord account connection, start.gg has their discord usernames. I can make the script so it also shows the player's discord name so I know what to ping when they're not checking in.

I could theoretically go farther and make this a whole discord bot, but then this'll be like, the 3rd bot I know of to do this same thing, and if it was actually good then it would have been more widely adopted by now.
