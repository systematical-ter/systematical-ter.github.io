So....

I have a blog now, and that's pretty neat! It's made using Pelican, and I have a script that takes my (private) source repository, compiles it, and pushes the output to the (public) repository that actually gets automatically made into a github pages website. That part was all fine and dandy!

Using Visual Editors

I decided to make life difficult for myself, though, and wanted to be able to edit the content using Obsidian, a little application I use when I need to write a lot of markdown files for notes or a guide or something. That was a little bit annoying, because I do my python coding in WSL, so the repository containing my blog .md is on the WSL file system. Trying to just point the windows Obsidian install to that folder caused a bunch of file read errors, so I went ahead and installed Obsidian on WSL itself.

I downloaded the .deb file, installed it, and then futzed around making it so my WSL2 install could actually run visual applications (because what's the point of all of this if I can't run linux visual apps, right?) Turns out I had to run a quick:

export DISPLAY=:0

... and it magically started working. Neat! (I probably have to do this every time I start a new terminal until I figure out where it's setting $DISPLAY to anything else.)

... And it was at this point I realized... I could probably just use VSCode instead. Oh well! I'll probably just use that next time and nuke Obsidian off my WSL, since it keeps trying to make folders in my root directory and I don't want it to do that.
