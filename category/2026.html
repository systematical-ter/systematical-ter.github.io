SteamSneggy

I technically started this bot second but also I finished it first so I'll talk about it first. It's stupid simple. If you're reading this you might know about a bot called "ShuuBox" that used to exist; basically it just watches for any messages that contain a steam lobby URL and responds to that message with the same URL, but now clickable. Well, that bot went offline a few months ago and that was kinda it.

So, I decided to make my own. Shouldn't be too hard, right?

(For once, it wasn't.)

The bot's stupid simple. The source code is available here. There's two components: the discord bot and the web API.

Discord Bot

I coded it in Python because that's what I'm comfortable with. There's a fairly good (I'll talk about my issues with it later) library in Python for communicating with Discord called discordpy, more information over here.

Deadass all it does is look for discord messages that fulfill a regex that looks for steam URLs, breaks it up into the three IDs, and makes a new URL that'll point to the web API, and sends a new message responding to the original message. It also mentions the person who originally wrote the message, and also pings anyone that they replied to or mentioned.

Simple. Just gotta make sure to enable message intent in the bot's information page, and note that the moment the bot hits 100 servers you/I'll be forced to hand Discord a government ID to keep the bot running because of that. We'll burn that bridge when we get there -- for now I'm just restricting who can install my bot.

Web API

The web API is the component I wasn't actually sure how to do at first, but turns out it's stupid simple too. All it needs to do is at a given url, redirect the requestor to the equivalent steam URL. I kept it as simple as possible -- there's a single endpoint, it takes the 3 IDs as url params, and given those 3 IDs redirects the user to a reconstructed steam lobby join link.

I specifically broke the steam lobby link into the three IDs when taking it as an input for a simple reason: I didn't want to have to deal with string safety lmao. If had the API just take whatever URL it was handed and redirect the user, that can get super sketchy super fast. If I try to do any fancy parsing, I run into problems with string parsing. So, the API just wants three integers, and the bot takes them from a discord message's content (which should be pre-sanitized by Discord to be safe.)
